# メタデータの表示、追加、書き出し{#viewing-adding-and-exporting-metadata}

使用するファイルに固有の情報をAdobe Dynamic Media Classicに保存することができます。この情報を次のように呼びます *メタデータ*. Adobe Dynamic Media Classicでメタデータを使用して、アセットの整理、検索、フィルタリングおよび並べ替えを行うことができます。

詳細ビューにメタデータが表示されます。 Adobe Dynamic Media Classicで生成された情報と共に表示されます。 例えば、ファイルの作成日、公開日、キーワードなどです。 メタデータを表示するには、詳細表示でアセットを開き、メタデータパネルを選択します。 詳細表示でメタデータを入力および編集できます。

一部のメタデータは、直接ファイルに埋め込まれます。ファイルにこのメタデータが含まれている場合、Adobe Dynamic Media Classicはファイルと共にメタデータを自動的にアップロードします。 Adobe Photoshop、InDesign、Illustrator、その他のアプリケーションのソースアセットにメタデータを埋め込むことができます。Adobe Dynamic Media Classicでは、このメタデータを認識します。 詳細表示のメタデータパネルで、個々のファイルにメタデータを追加することもできます。 アセット間で統一をとるために、会社の管理者は、入力できるメタデータフィールドを指定したメタデータテンプレートを作成できます。

埋め込みメタデータの詳細については、を参照してください。 [拡張可能なメタデータプラットフォーム](https://www.adobe.com/products/xmp.html).

## メタデータの表示 {#view-metadata}

アセットのメタデータを表示するには、詳細表示でアセットを開き、メタデータパネルをタップします。 一連のメタデータフィールドを選択するには、メタデータ表示メニューでオプションを選択します。 Adobe Dynamic Media Classicには、次のメタデータビューがあります。

* **コンパクト表示**：値の基本的なリスト。

* **IPTC**：国際プレス通信審議会で定義された値。

* **XMP**：拡張可能なメタデータプログラムで定義された値。

管理者はメタデータビューを作成できます。これらのビューは、メタデータビューメニューにも表示されます。

参照： [メタデータビュー](application-setup.md#metadata_views) メタデータビューの作成について

## アセットのメタデータの手動入力 {#manually-enter-metadata-for-an-asset}

1. アセットを詳細表示で開きます。
1. メタデータパネルを開いて、次のいずれかの操作またはその両方を実行します。

   * メタデータビューを選択して、パネルに表示するメタデータフィールドを指定します。
   * 「プリセット値」を選択し、 **[!UICONTROL 適用]** メタデータフィールドにプリセット値を入力します。 これらのプリセット値は、会社の管理者が作成します。

1. メタデータパネルに値を入力します。

>[!NOTE]
>
>複数のアセットのメタデータを一度に編集するには、アセットを選択し、に移動します。 **[!UICONTROL ファイル]** > **[!UICONTROL 情報の編集]**. 情報を編集ウィンドウでメタデータに対して行った編集は、選択したすべてのアセットに適用されます。

## キーワードの追加または編集 {#add-or-edit-keywords}

メタデータに加えて、キーワードを使用してアセットの検索と管理に役立てることができます。

このセッションで他のファイルにキーワードを追加した場合、またはリストからキーワードを削除した場合、「キーワード候補」 テーブルに表示されます。

1. 詳細ビューでファイルを開きます。
1. を選択 **[!UICONTROL キーワード]**.
1. キーワードを追加するには、次のいずれかの操作を行います。

   * テキストボックスにキーワードを入力して、 **[!UICONTROL 追加]**.
   * 内のキーワードを選択 **[!UICONTROL キーワード候補]** テーブル。

1. キーワードを削除するには、キーワードを選択し、 **[!UICONTROL 削除]**. そのキーワードがキーワード候補テーブルに移動します。

>[!NOTE]
>
>Adobe Dynamic Media Classicにファイルをアップロードする際に、ファイルにキーワードを追加できます。 アップロードジョブオプション ダイアログボックスで、を選択します **[!UICONTROL その他のメタデータ]** キーワードを入力します。
>詳しくは、[ アップロードオプション](uploading-files.md#upload_options)を参照してください。

## メタデータの取り込み {#import-metadata}

メタデータを 1 つずつ手動で入力する代わりに、タブ区切りファイルまたは XML ファイルから数多くのアセット用のメタデータを取り込むことができます。タブ区切りファイルまたは XML ファイルにメタデータを入力してファイルを取り込む方法の方が、個々のアセットにメタデータを入力する方法より手間が省けます。タブ区切りファイルの最初の行に、ID、およびメタデータを記録するフィールドの名前を入力します。後続の各行に、アセット ID 名に続けてメタデータ値を入力します。タブ区切りファイルまたは XML ファイルに含まれていないフィールドは変更されません。XML ファイルからメタデータを取り込むには、DTD に準拠していることを確認します。

>[!NOTE]
>
>メタデータを入力するためのテンプレートを作成して、Adobe Dynamic Media Classicに適切に読み込むことができます。 作成したテンプレートは、メタデータの入力に使用できます。
>詳しくは、[アップロードするメタデータを入力するためのテンプレートの作成](viewing-adding-exporting-metadata.md#create_a_template_for_entering_metadata_to_upload)を参照してください。

標準化されたプロパティの詳細については、次のページを参照してください [Adobe XMP デベロッパーセンター](https://www.adobe.com/devnet/xmp.html).

1. 参照パネルで、タブ区切りファイルまたは XML ファイルからメタデータを追加する画像を選択します。
1. に移動 **[!UICONTROL ファイル]** > **[!UICONTROL メタデータの読み込み]**.
1. が含まれる **[!UICONTROL メタデータのアップロード]** ダイアログ ボックスで、を選択します。 **[!UICONTROL 参照]**.
1. **[!UICONTROL アップロードするファイルを選択]**&#x200B;ダイアログボックスで、メタデータが含まれているタブ区切りファイルまたは XML ファイルを選択します。
1. ジョブ名を入力します。
1. を選択 **[!UICONTROL Upload]**.

### 読み込みでの様々なメタデータタイプの識別

読み込もうとする複数のメタデータの種類を識別する場合は、以下のことを考慮してください。

* ユーザー定義フィールドの名前は、で作成されたものとして識別されます。 **[!UICONTROL 設定]** > **[!UICONTROL アプリケーション設定]** > **[!UICONTROL メタデータ]** > **[!UICONTROL ユーザー定義フィールド]**. の使用 `Generate file` 正しいインポートフォーマットで定義されたすべての UDF のリストを取得する機能。
* XMP メタデータプロパティには、（プロパティ）名より前に、関連する XMP プリフィックスを付ける必要があります。プリフィックスと名前をコロンで区切ります。XMP プレフィックスは次の場所にあります。 **[!UICONTROL 設定]** > **[!UICONTROL アプリケーション設定]** > **[!UICONTROL メタデータ]** > **[!UICONTROL メタデータスキーマ]** 編集者。 技術的な名前は、関連する XMP スキーマのドキュメントで確認できます。XMP プロパティ名がに表示されない `Generate file` 機能
* メタデータスキーマプロパティには、（プロパティ）名より前に、関連するプリフィックスを付ける必要があります。プリフィックスと名前をコロンで区切ります。プレフィックスとプロパティ名は、メタデータスキーマエディターで定義されます。 メタデータスキーマのプロパティ名がに表示されない `Generate file` 機能

例：キーワードのXMP プロパティは、プレフィックスが付いたXMP スキーマ「Dublin Core」です `dc` および `subject` は、技術的なXMP名です。 プレフィックスと技術的なXMP名は、に結合されます。 `dc:subject` 完全なプロパティ名。 XML メタデータのインポート形式 `dc.subject` プロパティ名にする必要があります。 タブ区切りのインポートフォーマットでは、列ヘッダーである必要があります。

### キーワードの読み込み

キーワードは、コンマ区切りリストとしてインポートできます。 個々の値にコンマが含まれる場合は、バックスラッシュ（\）を使用してエスケープします。 円マーク自体を表すには、2 つの円マーク（￥￥）を使用します。

例えば、の値を含んだメタデータインポートファイルなどです `Hello\, World!,back\\slash,foo` （用） `dc:subject` は、アセットに 3 つのXMP キーワードを設定します。 `Hello, World!,` `back\slash,` および `foo`.

## メタデータの読み込み（FTP 経由） {#import-metadata-via-ftp}

複数のファイルのメタデータを読み込むことができます。 メタデータは、タブ区切り形式または XML ファイルで入力します。 次に、を選択します **[!UICONTROL メタデータファイルの処理]** （「FTP を使用」タブの） アップロードジョブオプション ページで、次の設定を行います。

タブ区切りファイルまたは XML ファイル内のデータが正しい形式になっていることを確認してください。最初の行に、ID フィールドに続けて、変更するメタデータフィールドの名前を入力します。後続の各行に、アセット ID 名に続けてメタデータ値を入力します。タブ区切りファイルまたは XML ファイルに含まれていないフィールドは変更されません。

グローバルナビゲーションバーで、を選択します。 **[!UICONTROL Upload]**. メタデータをインポートするには、アップロード ページで、 **[!UICONTROL FTP 経由]** tab キーを押してから、 **[!UICONTROL ジョブオプション]**. アップロードジョブオプション ダイアログボックスで、 **[!UICONTROL ジョブ]**&#x200B;を選択してから、 **[!UICONTROL メタデータファイルの処理]** チェックボックスをオンにします。

## メタデータを使用して ID をバッチで名前変更 {#batch-rename-ids-using-metadata}

タブ区切りファイルまたは XML ファイルから読み込まれたメタデータを使用して、Adobe Dynamic Media Classic ID の名前を変更できます。 取り込まれたメタデータは、メタデータファイルで指定されている画像だけに適用されます。参照パネルで画像が選択されているかどうかは関係ありません。

画像のAdobe Dynamic Media Classic ID の名前を変更するには、というラベルの付いた列を追加します *newipsid* タブ区切り形式のファイルに追加するか、というフィールドを追加します。 `new_vc_objectname` を XML データに追加します。

以下に例を挙げます。

| | newipsid |
| --- | --- |
| testjacket_1 | Jacket_test_1 |
| testjacket_blue | Jacket_test_2 |

メタデータ ジョブのジョブ ログには、名前が正常に変更された ID と変更されなかった ID が表示されます。

## アップロードするメタデータを入力するためのテンプレートの作成 {#create-a-template-for-entering-metadata-to-upload}

Adobe Dynamic Media Classicには、メタデータを記録するためのテンプレートを作成するためのコマンドが用意されています。 テンプレートを使用すると、メタデータをAdobe Dynamic Media Classicに正しくアップロードできるよう、適切な形式で入力されます。 メタデータの記録とAdobe Dynamic Media Classicへの読み込みに使用するテンプレートを作成するには、次の手順に従います。

1. テンプレートに必要なメタデータフィールドを持つ画像アセットを選択します。
1. に移動 **[!UICONTROL ファイル]** > **[!UICONTROL メタデータの読み込み]**.
1. の場合 **[!UICONTROL アセットプロパティタイプ]**&#x200B;を選択 **[!UICONTROL 画像]**.
1. から **[!UICONTROL `Generate File`]** ドロップダウンリストから次を選択します **[!UICONTROL タブ区切りテンプレート]**, **[!UICONTROL アセットの XML メタデータ]**、または **[!UICONTROL XML DTD]**.
1. を選択 **[!UICONTROL Generate]**.
1. 表示されるダイアログボックスで、データをコピーします。このデータを使用してテンプレートを作成します。

## メタデータスキーマの操作 {#working-with-metadata-schemas}

会社管理者は、使用可能なすべてのスキーマのリストを表示できます。 グローバルナビゲーションバーで、に移動します。 **[!UICONTROL 設定]** > **[!UICONTROL アプリケーション設定]** > **[!UICONTROL メタデータ]** > **[!UICONTROL メタデータスキーマ]**.

最初は、XMPなどのグローバル標準スキーマのリストは非表示になっています。 リストの下部にあるチェックボックスを使用してこれらを表示できます。

会社管理者は、カスタムスキーマを作成したり、既存のカスタムスキーマを編集したりできます。

メタデータスキーマエディタを使用して以下のアクションを実行できます。

| アクション | 説明 |
| --- | --- |
| 追加 | スキーマにプロパティを追加します。 モーダルダイアログは、情報（ID、ラベル、構造、データタイプ）を収集します。 |
| 選択値を追加 | 自由選択、または選択肢から選択の構造を使用して、選択可能な新しい選択肢をプロパティに追加します。すべての選択値は同じタイプを持ちます。プロパティ自体を選択して、ボタンを有効にします。 |
| 編集 | プロパティまたは選択値のラベルを編集します。変更できるのはラベルのみであり、ID とタイプの情報は変更できません。 |
| 上へ移動/下へ移動 | スキーマ内の順序は、UI に反映されます。順序を変更するには、プロパティまたは選択値を選択し、ボタンを使用して移動します。ドラッグ&amp;ドロップは、現在サポートされていません。 |
| 削除 | スキーマからプロパティまたは選択値を削除します。XMP ブロックやデータベースからは値が削除されません。 このプロパティは、メタデータビューでは使用できなくなり、アセットの詳細ビューから削除されます。 プロパティが Metadata Server に公開されている場合は、[ 強制公開 ] を実行して、公開されている Metadata Server からデータを削除します。 |

プレフィックスがのユーザー定義フィールドのカスタムスキーマが自動的に生成されます `s7udf`. スキーマは、独自の設定セクションで編集される既存のユーザー定義フィールドで構成されます。

>[!NOTE]
>
>スキーマを変更しても、アセットのメタデータ自体は決して変更されません。ただし、Adobe Dynamic Media Classicおよびメタデータサーバーのすべての機能では表示されず、変更後にアクセスすることはできません。 同様に、アセットのメタデータが存在する場合、一致するスキーマが作成されると、Adobe Dynamic Media Classicと Metadata Server でメタデータを使用できるようになります。

メタデータスキーマエディターを使用すると、Adobe Dynamic Media Classic内でカスタムの会社スキーマをグラフィカルに追加または編集できます。 プレフィックス、名前空間、プロパティのリストは、スキーマを定義します。

* **[!UICONTROL 名前]**：スキーマの UI 名。 メタデータビューおよび詳細検索でプロパティを識別するために使用します。基本、IPTC、PDF のような XMP セクションに似ています。

* **[!UICONTROL プレフィックス]**：スキーマの技術的な一意の ID。 a～z と A～Z の文字に制限されます。このプレフィックスは、Adobe Dynamic Media Classic UI には表示されませんが、アセットのメタデータがXMP ブロックとデータベースに格納される際に使用されます。 このプレフィックスにより、メタデータ サーバーまたはインポートのメタデータ検索クエリのメタデータ フィールドが一意に識別されます。

* **[!UICONTROL 名前空間]**：スキーマの技術的な一意の識別子（通常はフォーム内の URL） `https://your.company.com/name/version/`. 標準スキーマリストで例を参照してください。名前空間は、Adobe Dynamic Media Classic UI には表示されませんが、XMP ブロックにメタデータを保存するために使用されます。

* **[!UICONTROL 説明]**：スキーマに関する自由形式の説明。

>[!NOTE]
>
>プリフィックスと名前空間は編集できません。これらのプロパティを変更するには、スキーマを削除し、もう一度作成する必要があります。

プロパティは、このスキーマとともに XMP ブロックに格納できるメタデータを記述します。プロパティは以下のもので構成されます。

| プロパティ | 説明 |
| --- | --- |
| ID | このプロパティの技術的な識別子。この ID は、Adobe Dynamic Media Classic UI には表示されませんが、アセットのメタデータがXMP ブロックとデータベースに格納される際に使用されます。 この ID は、Metadata Server で検索クエリを作成するために使用されます。 ID には、次のような制限があります。 `<ul><li>No spaces</li><li>No ".", ":", "$"</li><li>No number as first character</li><li>Best practice is to use a-z or A-Z as first character</li></ul>` <br>作成後は、ID を変更できません。 |
| ラベル | このプロパティの UI 名。 |
| 構造 | データタイプとともに、プロパティのタイプを決定します。構造は次のいずれかです。<ul><li>シンプルタイプ：データタイプの単一の値</li><li>シーケンス：同じデータタイプを持つ複数の値から構成されるリスト</li><li>選択肢を開く：事前定義済みの値のリストから 1 つの項目を選択するか、テキストを入力します。 データ型は、文字列または整数のみです</li><li>選択肢から選択：定義済みのリスト（ポップアップまたはコンボボックス）からいずれかの項目を選択</li></ul> |
| データタイプ | 以下の使用可能なタイプから選択します。 <ul><li>文字列</li><li>整数</li><li>浮動小数点</li><li>はい／いいえ（ブール値）</li><li>日付</li></ul> |

プロパティの構造が「選択肢を開く」または「選択肢を閉じる」の場合、少なくとも 1 つの選択肢の値を指定する必要があります。 自由選択では変更が可能です。選択肢から選択では変更が不可能です。すべての選択値には、プロパティのデータタイプがあります。

| プロパティ | 説明 |
| --- | --- |
| ID | この値の技術的な識別子。この ID は、Adobe Dynamic Media Classic UI には表示されませんが、アセットのメタデータがXMP ブロックとデータベースに格納される際に使用されます。 ID は、メタデータサーバに対する検索クエリの中で使用されます。ID にスペースを含めることはできません。ID は作成した後に変更できません。 |
| ラベル | この値の UI 名。 |
